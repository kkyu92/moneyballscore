## Decisions (결정 내용 및 이유)

### next → 16.2.6 (16.2.5 아닌 이유)
마지막 취약점(Middleware bypass incomplete fix follow-up)이 `>=16.2.6` 을 요구. 16.2.6 = 현재 최신 stable.

### pnpm overrides vs 직접 업그레이드 (Sentry 등)
`fast-uri`, `postcss` 는 Sentry/webpack 의 deep transitive dep (5층 이상). 상위 패키지 업그레이드는 호환성 위험 > 이익. pnpm overrides 로 최소 변경 해결.

### CI audit 레벨 `--audit-level=high`
- `critical` → 너무 낮은 기준 (아직 critical 없음)
- `moderate` → postcss transitive 등 false positive 많음
- `high` → Next.js DoS/bypass/SSRF 류를 모두 잡는 적절 임계

### `actions/checkout`, `actions/setup-node` SHA 핀 생략
GitHub 자사 액션 (actions/*). 공급망 공격 주체가 GitHub 자체여야 가능 → risk/benefit 비율 낮음. 제3자 `pnpm/action-setup` 만 핀.
